FraudShield AI — Technical Deep Dive
AI/ML Course · Jaypee Institute of Information Technology, Noida · Dr. Niyati Agrawal · Batch B3

FraudShield is a complete end-to-end machine learning system that detects credit card fraud in real time. It ingests raw transaction data, trains and compares multiple classifiers, selects the best model, and exposes it via a Flask REST API with this interactive analytics dashboard on top.

The core challenge is extreme class imbalance: only 0.172% of 284,807 transactions are fraudulent (492 cases). Standard accuracy is meaningless here — a model predicting everything as legit gets 99.83% accuracy but catches zero fraud. We solve this with class-weight balancing and evaluate using AUPRC, the only metric that stays honest under severe imbalance.

📊 Dataset — Kaggle ULB Credit Card Fraud
Source
European cardholders, September 2013 (2-day window)
Size
284,807 transactions × 31 columns
Fraud Cases
492 (0.1727%) — severe class imbalance
Features
V1–V28 (PCA-transformed), Time (seconds since first tx), Amount (EUR)
Target
Class: 0 = Legitimate, 1 = Fraud
Privacy
V1–V28 are PCA-transformed to protect cardholder identity — original feature names undisclosed by dataset authors
🔢 Variable Explanations — Every Feature, In Detail
VariableTypeWhat It Is & How We Use It
TimeRaw Seconds elapsed since the first transaction in the dataset (~48-hour window). Raw values reach 172,792 — without scaling this single feature would dominate any distance-based calculation. We apply StandardScaler (mean=0, std=1). Why keep it? Fraud clusters by hour — our dashboard shows peaks at midnight and midday — so time genuinely adds predictive signal.
AmountRaw Transaction value in Euros. Fraud avg = €122.21 vs legit avg = €88.29. Fraudsters often test cards with moderate amounts before larger withdrawals. Also StandardScaled. Critically, the scaler is fit on training data only then transform-only on test — preventing data leakage where test-set statistics would contaminate training.
V1 – V28PCA 28 principal components computed by the dataset authors from the original confidential features (merchant category, device fingerprint, card type, location, etc). Each Vi is a linear combination of original features pointing in a direction of maximum variance. PCA was applied at source to anonymise the data. V14, V10, V17, V12 rank highest in importance — strongly negative values in these components are the clearest markers of fraudulent patterns in this dataset.
ClassTarget Binary ground-truth label: 0 = legitimate, 1 = fraud. Used only as training signal and evaluation reference — never as model input. We stratify the train/test split on this column so both subsets maintain the same 0.17% fraud rate, preventing one split from accidentally having more or fewer fraud cases.
🌲 Why Random Forest? (Implementation Rationale)
✦
Ensemble of 100 decision trees — each tree trains on a random bootstrap sample with a random feature subset (√30 ≈ 5 features per split). Final fraud probability is the average across all trees, making it robust to noise typical in financial transaction data.
✦
class_weight='balanced' — automatically assigns each fraud sample a weight of ~578× a legit sample (inversely proportional to class frequency). Without this, the model minimises loss by predicting everything legit, achieving 99.8% "accuracy" while catching zero fraud.
✦
Non-linear decision boundaries — fraud patterns like "V14 < −8 AND Amount > 500 AND Time between 02:00–04:00" are conjunctive rules a linear model misses entirely. Decision trees capture these naturally without manual feature engineering.
✦
Built-in feature importance — RF computes how much each feature reduces Gini impurity across all trees, giving explainability into which PCA components drive predictions — critical when every block decision must be justifiable to a cardholder.
✦
Outperforms Logistic Regression — LR achieves AUPRC ≈ 0.71, RF achieves 0.8425. The non-linearity advantage yields +13 percentage points on the most honest metric for this dataset.
🧠 ML Concepts — Detailed Explanations
Supervised Learning

We train on labeled data where each row has Class=0 or 1. The model learns f(V1…V28, Time, Amount) → P(fraud). 80% of data is training, 20% held out for unbiased evaluation. Stratified split ensures equal fraud rate in both subsets.

Class Imbalance Handling

Three simultaneous strategies: (1) class_weight='balanced' in RandomForestClassifier, (2) stratify=y in train_test_split, (3) AUPRC as the primary evaluation metric rather than accuracy. Together they prevent the model from exploiting the 99.83% majority as a shortcut.

Feature Scaling

StandardScaler: (value − mean) / std → mean=0, std=1. Applied to Amount and Time only (V1–V28 are pre-scaled from PCA). Fit on X_train only, then transform-only on X_test — no data leakage.

AUPRC — The Right Metric

Area Under Precision-Recall Curve. A random classifier gets AUPRC ≈ 0.0017 (= fraud rate). Our model gets 0.8425 — ~490× better than random. AUROC inflates artificially under imbalance because the huge number of True Negatives makes TPR look great even for poor fraud detection. AUPRC only rewards correctly catching actual fraud.

PCA Features (Pre-applied)

V1–V28 are already-computed principal components. PCA finds orthogonal directions of maximum variance. V14 being most important means the 14th principal component captures the direction in feature space that best separates fraud from legit — likely encoding merchant + device + location patterns simultaneously.

⚡ API Endpoints — How They Work
GET
/api/model-data

Returns all pre-computed analytics as a single JSON blob: model results, confusion matrices, ROC/PR curve arrays, feature importances, hourly histogram, dataset stats. Called once on page load. Pre-computed at train time — API latency under 5 ms regardless of dataset size.

POST
/api/predict

Accepts JSON with V1–V28, Time, Amount. Scales Amount and Time using the saved scaler, builds a 1-row DataFrame, runs predict_proba() on the saved RF model, returns fraud_probability, risk_level (LOW/MEDIUM/HIGH), and verdict string. Latency <15 ms.

POST
/api/batch-predict

Accepts {"transactions":[{...}]}. Vectorised — builds one DataFrame for all transactions and calls predict_proba once. Returns per-transaction scores plus a flagged count. Scales linearly with batch size; no per-row overhead.

GET
/api/stats

Returns dataset-level aggregates: total count, fraud count, fraud percentage, amount statistics for fraud vs legit. Used by the Overview metric cards on page load.
